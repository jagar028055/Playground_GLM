# タスク判定チャート：boss1に相談すべきか？

**対象**: President
**目的**: 5分でboss1活用の可否を判定
**更新**: 2026-01-31

---

## 🎯 クイック判定（5分で回答）

### ステップ1：所要時間の見積もり

```
このタスクにかかる時間は？

A. 30分以内     → 【レベル1】単独実行
B. 30分〜2時間   → 【レベル2】boss1に相談
C. 2時間〜1日    → 【レベル3】boss1に委任
D. 1日以上       → 【レベル4】boss1+workersで実行
```

### ステップ2：作業の複雑さ

```
複数の種類の作業が必要か？

A. いいえ（単一作業のみ）
   → 単独実行を検討

B. はい（調査＋分析＋作成など）
   → boss1に委任を検討
```

### ステップ3：専門性の有無

```
専門的な判断や分析が必要か？

A. いいえ（既定のプロセス）
   → 単独実行可能

B. はい（新しい判断が必要）
   → boss1に相談
```

## ✅ 判定結果

### パターン1：単独実行（boss1不要）

**条件**: 以下の全てに当てはまる
- [ ] 30分以内で完了できそう
- [ ] 単一の作業で完了
- [ ] 既定のプロセスがある
- [ ] 失敗しても影響が軽微

**例**:
- 既存のレポートに数値を追加する
- スケジュールの確認
- ステータスチェック
- 簡単なメール返信

**アクション**:
```
すぐに実行開始
```

### パターン2：boss1に相談（実行はPresident）

**条件**: 以下のいずれかに当てはまる
- [ ] 30分〜2時間かかりそう
- [ ] 2-3種類の作業が必要
- [ ] 新しい判断が必要
- [ ] 少し複雑な依存関係がある

**例**:
- 既存プロセスの改良
- 複数ソースの情報統合
- 小規模なドキュメント作成
- 複数の選択肢の比較

**アクション**:
```bash
./agent-send.sh boss1 "【相談】

タスク: [タスク名]
背景: [なぜこのタスクが必要か]
想定時間: [X]時間

検討中のアプローチ:
1. [アプローチ1]
2. [アプローチ2]

アドバイスをお願いします。30分以内に返信を。"
```

## 🔍 よくある質問

### Q1: 30分かかるかどうか判断が難い場合

**A**: 安全サイドで見積もり、×1.5倍した時間で判定してください。

```
例: 20-30分かかりそう → 45分で判定 → レベル2（相談）
例: 1時間かかりそう → 1.5時間で判定 → レベル2（相談）
例: 3時間かかりそう → 4.5時間で判定 → レベル3（委任）
```

### Q2: 緊急度が高い場合はどうするか

**A**: 緊急度が高いほど、boss1に委任すべきです。

```
緊急度: 高 → boss1+workersで並行作業
緊急度: 中 → boss1に委任
緊急度: 低 → 時間があるときに実施
```

### Q3: 失敗できない重要なタスクはどうするか

**A**: 重要度が高いほど、複数人の視点が必要です。

```
重要度: 高 → boss1+workersで実施（複数の視点）
重要度: 中 → boss1に委任
重要度: 低 → 単独でOK
```

### Q4: boss1もworkersも忙しそうな場合はどうするか

**A**: まずboss1に相談し、優先順位を決めてもらってください。

```bash
./agent-send.sh boss1 "【優先順位相談】

新しいタスク: [タスク名]
想定時間: [X]時間
緊急度: [高/中/低]

現在の進捗状況を教えてください。
このタスクはいつから着手できそうですか？"
```

## 💡 プラクティス

### 良い習慣

1. **タスク受領時に5分で判定**
   - 直感でOK、迷ったらboss1に相談

2. **曖昧さは相談に変換**
   - 「どうすればいいか分からない」→ boss1に相談

3. **小さなタスクから委任開始**
   - 最初は1時間程度のタスクから

4. **失敗を恐れない**
   - 完璧主義は委任の障壁

### 避けるべきこと

1. **「自分でやった方が早い」の罠**
   - 短期的には正しいが、長期的には組織の能力が育たない

2. **「説明する時間が惜しい」の罠**
   - テンプレートを使えば効率化できる

3. **「品質が心配」の罠**
   - 明確な基準とレビューで解決
